# Dev log — 2026-07-13: from empty repo to full survival game

One Claude Code session took this project from an empty folder to the complete
game currently on `main`. The raw machine-readable transcript of the whole
session is in [`session-2026-07-13-full-build.jsonl`](session-2026-07-13-full-build.jsonl)
(Claude Code session format: one JSON event per line — user messages, assistant
messages, tool calls and results).

## What got built, in order

### Stage 1 — core survival loop
- Node + `ws` server (no framework), vanilla-JS canvas client, `shared/` defs
  imported by both sides. All art procedural.
- Gathering (trees/rocks/bushes, 3-min respawns, ARK-style tool affinity),
  crafting (stone axe/pick/spear), thatch building (foundation/wall/doorway/
  roof with support rules), campfire + cooking, shared storage boxes,
  hunger/starvation, day/night cycle, chat, 8-player multiplayer with
  per-name save profiles.

### Stage 2 — first dinos
- Dodos: wander/flee AI, hunting for meat + hide, ARK-style passive taming
  with berries, tamed follow/stay (T), egg laying.

### Adversarial review round 1
- Two multi-agent review passes (25 agents total) confirmed **21 real bugs**,
  all fixed. Highlights: a critical one-shot-iterator bug that silently
  disabled all building support rules; HiDPI mouse math off by 1/dpr;
  per-browser identity tokens so a name can't be hijacked; ws heartbeat,
  rate limiting, payload caps, atomic saves, prototype-pollution guards.

### Expansion (this session's second half)
1. **Terrain** — deterministic heightmap (`shared/terrain.js`) with hills,
   valleys, and streams; slope physics; five regions (`shared/regions.js`)
   from safe Spawn Meadow to Scorched Badlands; metal veins seeded.
2. **Combat + roster** — two-way combat with knockback/damage numbers/screen
   shake; compy, dilo, parasaur (rideable mount, R), raptor (needs full metal
   armor), T-Rex (avoid). Threat-colored name labels.
3. **Portals** — hub in the meadow with danger-colored portals to each region
   entrance + return portals home. Boot-time fixtures, never saved.
4. **Night re-randomization** — unoccupied chunks wipe + re-roll nodes and
   dinos each night; bases/portals/tames preserved.
5. **Metal tier** — forge smelting (ingots + charcoal), metal tools (1.7x),
   full armor set (up to 78% reduction — the raptor gate), gunpowder,
   bullets, rifle with tracers and ammo.
6. **Thirst** — drains faster than hunger; drink at streams (E) or eat
   berries; blue water bar.
7. **ESC options** — shared world rules (hunger/thirst/damage toggles, day
   length) + local mute + quit.
8. **Procedural sound** — every effect synthesized with WebAudio at runtime
   (chops, clinks, gunshots, growls with distance falloff, portal warps).
   No audio files. M mutes.

### Player-reported fixes (live playtesting during the session)
- **Death loop**: HP regen ran before the death check, reviving players at
  0 HP every tick — players got chewed forever instead of dying. Fixed and
  regression-tested (bot walked into a rex, died properly, respawned).
- **Invisible build costs**: requirements were tooltip-only; now every build
  button shows its full material list, red for missing.

## How it was tested
- The project owner played live in a browser throughout (their base at
  x≈5900 in the flat-world era was the first real QA pass).
- Headless WebSocket bots: multiplayer sync, full 8-feed tame, hunting,
  cooking, portals round-trip, combat/death regression, thirst/settings,
  hostile-input security (floods, 100KB frames, `__proto__` names).
- Direct handler-import unit tests: placement rules, night re-rolls, the
  full smelt→craft→wear→shoot chain.
- Two adversarial multi-agent review workflows with independent verification
  of every finding before fixing.

## Where things stand
Everything above is on `main` and playable at `http://<host>:3000`. The
second expansion review workflow's confirmed findings (if any) land as the
commits immediately after this file. Next candidates: saddles + more mounts,
